# Security Policy

## Supported Versions

We are committed to providing security updates for the current stable release and the previous long-term support version. Please ensure you are using a supported version to remain protected.

| Version | Supported          | Status |
| ------- | ------------------ | ------ |
| 1.1.x   | :white_check_mark: | Current Stable Release |
| 1.0.x   | :white_check_mark: | Maintenance Support |
| < 1.0   | :x:                | Legacy - Please upgrade |

---

## Reporting a Vulnerability

**Smart Link** takes the security of our digital assets and our partners' data seriously. If you believe you have found a security vulnerability, please follow the steps below:

### How to Report
Please **do not** open a public GitHub issue for security vulnerabilities. Instead, please report them via:

1.  **Email:** Send a detailed report to **info@smartlinkco.jp**.
2.  **Private Report:** Use the **"Report a vulnerability"** button under the **Security** tab of this repository.

### What to Include
To help us resolve the issue quickly, please include:
* A description of the vulnerability and its potential impact.
* Steps to reproduce the issue (proof-of-concept code or screenshots are helpful).
* Any specific environment details (browser, OS, etc.).

### Our Commitment
* **Acknowledgment:** We will acknowledge receipt of your report within **48 hours**.
* **Updates:** You will receive a status update every **3-5 business days** until the issue is resolved.
* **Resolution:** Once a fix is verified, we will coordinate a disclosure date with you. 

We appreciate your help in keeping Smart Link and our partners secure.
